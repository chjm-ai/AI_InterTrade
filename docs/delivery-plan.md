# 月底交付计划

## 交付目标

| 能力 | 最低交付物 | 建议补充 |
| --- | --- | --- |
| 多平台发布 | 功能截图 | 账号矩阵示意图、内容流转图 |
| 爆款内容拆解 | 功能截图 | 输入输出样例、分析报告样稿 |
| 多平台多语种询盘承接 | demo + 视频 | 渠道接入流程图、消息分配说明 |
| 评论截流 | 功能截图 | 风控说明、人工介入流程 |

## 执行顺序

1. 先做询盘承接 demo
2. 再跑爆款内容拆解
3. 然后补多平台发布截图
4. 最后决定评论截流是否继续投入

## 每个能力都要产出的材料

- 使用的第三方工具
- 账号/环境准备条件
- 配置步骤
- 成功截图或录屏
- 成本估算
- 风险与替代方案
- 是否值得继续投入

## 成功标准

不是“技术上理论可行”，而是满足以下条件：

- 能复现
- 能演示给客户看
- 能估一个初步报价
- 能说明后续如何接入自家平台
